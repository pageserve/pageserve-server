PageServe documentation

A self-hosted REST API around PageIndex OSS for vectorless document RAG: multi-tenant projects, JWT + API key-pair auth, an ARQ indexing queue, and the pageserve SDK. Your server:

Overview

pageserve-server turns a folder of PDFs into a question-answering API with page-level citations. Instead of chunk + embed + vector search, it uses an LLM-built document structure tree (a table of contents) and navigates that tree to fetch exactly the pages that answer a question — accurate, with traceable sources.

📁 Multi-tenant

Each project is an isolated space of documents and keys.

🔑 Two auth layers

JWT for the Admin UI, public/secret keys for agents & the SDK.

⚙️ Async indexing

ARQ queue, RAM-adaptive, retries, with live progress over SSE.

🧩 SDK + MCP

Use it as a Python library or plug it straight into an agent via MCP.

Get started

From install to your first query in 5 steps:

  1. Create a project on the Projects tab — give it a name and slug.
  2. Upload a PDF to the project. Its status moves through pending → indexing → completed.
  3. Wait for indexing to finish (tracked in real time in the UI).
  4. Create an API key on the API Keys tab — save the secret_key (shown only once).
  5. Query via the SDK, REST, or try it instantly in the Playground.

Your first query with curl:

curl -X POST __BASE__/v1/query \
  -u "pk_live_xxx:sk_live_xxx" \
  -H "Content-Type: application/json" \
  -d '{"doc_id": "DOC_UUID", "question": "Summarize chapter 2"}'

Build & deploy

The whole stack runs with Docker Compose. First time:

# 1. Configure
cp .env.example .env
openssl rand -hex 32        # paste into JWT_SECRET

# 2. Bring up everything (Postgres + Redis + API + Worker)
docker compose up -d

# 3. Follow the logs
docker compose logs -f pageserve
docker compose logs -f worker

Required environment variables in .env:

POSTGRES_PASSWORD=...             # database password
LLM_BASE_URL=http://vllm:8000/v1  # OpenAI-compatible backend
LLM_MODEL=Qwen/Qwen3.6-27B
ADMIN_EMAIL=admin@company.com
ADMIN_PASSWORD=...                # change right after first login
JWT_SECRET=...                    # openssl rand -hex 32

# Optional — leave empty to auto-detect from host RAM
MAX_FILE_SIZE_MB=
WORKER_MAX_JOBS=
RATE_LIMIT_PER_MINUTE=0

Local development (no Docker):

source .venv/bin/activate
pip install -r requirements.txt
# needs a running Postgres + Redis; export the same vars as .env
uvicorn app.main:app --reload      # API (runs migrations on boot)
arq worker.WorkerSettings          # indexing worker (separate process)

The API runs Alembic migrations and seeds the default admin on startup. Open for Swagger.

Authentication

Two surfaces, two mechanisms:

Admin UIAgent / SDK
CredentialsEmail + password → JWTpublic key + secret key
Endpoints/auth/* · /admin/*/v1/*
Tokenaccess 1h + refresh 7dverified per request

Pass the key pair via HTTP Basic (recommended) or two dedicated headers:

# Option 1 — HTTP Basic: base64("pk:sk")
Authorization: Basic cGtfbGl2ZV94eHg6c2tfbGl2ZV94eHg=

# Option 2 — separate headers (easier to debug)
X-PageServe-Public-Key: pk_live_xxx
X-PageServe-Secret-Key: sk_live_xxx

Scopes: read (list/get/query) and write (upload/delete/reindex/create key/webhook). The public key is safe to expose; if a secret key leaks, revoke it immediately.

Python SDK

Install:

pip install pageserve            # core SDK
pip install "pageserve[all]"     # + MCP server + CLI

Synchronous client — upload, query, read pages, inspect the structure tree:

from pageserve import PageServeClient

client = PageServeClient(
    base_url   = "__BASE__",
    public_key = "pk_live_xxx",
    secret_key = "sk_live_xxx",
)

# Upload and wait for indexing to finish
doc = client.upload("./report.pdf", wait=True)
print("Indexed:", doc.doc_id)

# Query with page-level citations
res = client.query(doc.doc_id, "What was Q4 revenue?")
print(res.answer)          # the answer
print(res.page_refs)       # [22, 24]
print(res.sources)         # "Annual Report 2024 p.22, 24"

# Read specific pages (no LLM call)
for p in client.get_pages(doc.doc_id, "22-24"):
    print(f"--- page {p.page} ---\n{p.content}")

# Structure tree / table of contents
for node in client.get_structure(doc.doc_id):
    print(node.title, f"(p.{node.start_index}-{node.end_index})")

Async client

Query multiple documents concurrently with AsyncPageServeClient:

import asyncio
from pageserve import AsyncPageServeClient

async def main():
    async with AsyncPageServeClient(
        base_url   = "__BASE__",
        public_key = "pk_live_xxx",
        secret_key = "sk_live_xxx",
    ) as client:
        results = await client.query_many([
            ("uuid-1", "probation terms"),
            ("uuid-2", "leave policy"),
        ])
        for r in results:
            print(r.doc_name, "→", r.answer[:80])

asyncio.run(main())

REST API

The main endpoints live under /v1 (key-pair auth). Full reference at .

MethodPathDescription
POST/v1/documentsUpload a PDF (multipart)
GET/v1/documentsList documents
GET/v1/documents/{id}/structureStructure tree
GET/v1/documents/{id}/pages/{pages}Page contents
POST/v1/queryRAG query — synthesized answer with citations
POST/v1/query/streamAnswer query as an SSE stream
POST/v1/retrieveRelevant original passages — no answer

query vs retrieve: /v1/query runs the full agent loop and returns a written answer + sources; /v1/retrieve runs one tree-navigation call per document and returns the raw page content of the relevant sections (cheaper, no synthesis). Use retrieve to feed passages into your own model or display source text.

Upload with curl:

curl -X POST __BASE__/v1/documents \
  -u "pk_live_xxx:sk_live_xxx" \
  -F "file=@report.pdf"

Retrieve relevant original passages (no answer):

curl -X POST __BASE__/v1/retrieve \
  -u "pk_live_xxx:sk_live_xxx" \
  -H "Content-Type: application/json" \
  -d '{"doc_id": "DOC_UUID", "question": "probation terms"}'

# → { "results": [ { "doc_id": "...", "sections": [
#       { "title": "Probation", "page_start": 24, "page_end": 25,
#         "pages": [ {"page": 24, "content": "..."} ] } ] } ] }

Webhooks

Register a URL to receive document.completed / document.failed events when indexing finishes. Payloads are signed with HMAC-SHA256 in the X-PageServe-Signature header.

# Verify the signature (Python)
import hmac, hashlib

def verify(secret: str, body: bytes, signature: str) -> bool:
    expected = "sha256=" + hmac.new(secret.encode(), body, hashlib.sha256).hexdigest()
    return hmac.compare_digest(expected, signature)
